PiLR Glossary

PiLR
A platform for mobile health researchers
Period
A defined interval for participants, possibly overlapping with other periods.
Epoch
A defined interval of time *within periods* for participants, not overlapping with other epochs in the same period
Participant
An individual enrolled in a study who data is able to upload data from instruments into PiLR
Instrument
Software or hardware device configured to upload data to PiLR
mHealth
Abbreviation for 'mobile health', the type of data PiLR is designed to collect
Project
A name for a study with participants, instruments, and settings
API
Abbreviation for 'application programming interface', allows PiLR to interact with instruments and dashboards
Dashboard
Set of panels displaying charts that summarize participant data for a project
Analysis Tool
Web apps designed to allow you to interact and explore your project data
Dataset
Instruments write to one or more datasets, which can be merged with each other
Template
A set of configuration rules for participants in the project, that defines what instruments are enabled during specific periods
Category
Grouping of participants by a category, such as gender or treatment
Group
Specific group within a category. If 'gender' is the category, ' male' and 'female' are the groups
Project Team
Assign roles to PiLR users for a project
Participant Data
Data generated by an instrument for a particular participant
EMA
Smartphone app to deliver versatile mobile surveys to participants
Setting
Cause instruments to behave differently depending on value
Schedule
Concrete realization of a template declaring exactly when certain events happen for a given group category or participant
